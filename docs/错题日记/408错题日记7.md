## 遍历顺序记混——前中后序的根 🔴

### 你的问题
"前序后序中序居然记混了"——还好看了书补回来了。

### 正解（一劳永逸的记忆法）

| 遍历 | 访问顺序 | 口诀 | 根的位置 |
|---|---|---|---|
| **前序** | **根** → 左 → 右 | 根在**前** | 第一个输出根 |
| **中序** | 左 → **根** → 右 | 根在**中** | BST 结果递增 |
| **后序** | 左 → 右 → **根** | 根在**后** | 最后一个输出根；常用于**释放空间** |
| **层序** | 逐层从左到右 | 用**队列**，BFS | — |

> **"根在哪，就叫什么序"**——左、右的相对顺序永远不变（先左后右），只有根的插入位置变。这是唯一的变量。

### 还原二叉树（必考）

| 给的条件 | 能否唯一还原 | 原因 |
|---|---|---|
| 前序 + 中序 | ✅ | 前序定根，中序分左右子树 |
| 后序 + 中序 | ✅ | 后序定根（最后一个），中序分左右 |
| 前序 + 后序 | ❌（一般） | 无法区分"只有左孩子"和"只有右孩子" |

> **识别信号**：题干给两种遍历序列 → 先从**前序首元素 / 后序尾元素**找根 → 在**中序**里切分左右子树 → 递归。

### 王道真题（RAG 提取）

**Q37（线索二叉树）**：中序遍历序列 `debxac`，结点 x 的左线索 = 中序序列中 x 左边的字符 = **b**，右线索 = 右边 = **a**。选 D。

> **线索 = 遍历序列中的前驱/后继**。写线索二叉树题，先把**对应遍历序列**写出来，直接从序列里读前驱后继。

## 线索二叉树——新概念，从零建框架 🟢

### 核心动机（RAG 提取）
$n$ 个结点的二叉树有 $n+1$ 个**空指针域**（$2n_0+n_1$，又 $n_0=n_2+1$ → $n+1$）。这些空指针**白白浪费**。线索化就是：**用空指针指向遍历序列中的前驱/后继**。

### 三种线索化

| 类型 | 空指针指向 | 适用 |
|---|---|---|
| **前序线索化** | 前序序列中的前驱/后继 | 找后继容易：有左孩子→左孩子；无→rchild 直接指 |
| **中序线索化** | 中序序列的前驱/后继 | 最常考；BST 中序线索 = 按递增顺序串成链 |
| **后序线索化** | 后序序列的前驱/后继 | 找后继最复杂（需知父结点） |

### 线索化标志位

```
ltag = 0 → lchild 指向左孩子
ltag = 1 → lchild 指向前驱（线索）
rtag = 0 → rchild 指向右孩子
rtag = 1 → rchild 指向后继（线索）
```

### 考点速查

| 考点 | 关键 |
|---|---|
| 空指针数 | $n+1$ 个（$n$ 个结点的二叉树） |
| 线索数 | $n+1$ 条（全部空指针变线索） |
| 中序线索找后继 | rtag=1 → rchild 直接读；rtag=0 → 右子树最左下角 |
| 前序线索找后继 | 有左孩子→左孩子；无→rchild 直接读 |

## 二叉树核心性质（王道高频选择）

| 性质 | 公式 | 易错点 |
|---|---|---|
| 度的关系 | $n_0 = n_2 + 1$ | 永远成立（非空二叉树） |
| 总结点 | $n = n_0 + n_1 + n_2$ | 完全二叉树 $n_1 \in \{0,1\}$ |
| 完全二叉树编号 | 第 $i$ 个：左孩子 $2i$，右孩子 $2i+1$，父 $\lfloor i/2 \rfloor$ | 从 1 开始编号 |
| 完全二叉树高度 | $\lfloor \log_2 n \rfloor + 1$ | — |
| 哈夫曼树 | 只有 $n_0$ 和 $n_2$（$n_1=0$），$n_2=n-1$（$n$ 个叶 → $n-1$ 次合并） | 前缀编码 |

## 森林 ↔ 二叉树转换（RAG 提取·2020 真题考点）

**转换规则**：**左孩子右兄弟**——第一棵树的子树 → 左子树，剩余森林 → 右子树。

| 树/森林遍历 | 对应二叉树遍历 |
|---|---|
| 树先根 / 森林先序 | 二叉树**先序** |
| 树后根 / 森林中序 | 二叉树**中序** |

> **记忆**：森林先序 = 二叉树先序，森林中序 = 二叉树中序。**森林没有"后序"**——后根遍历树对应的是二叉树中序。

## Day 2 补充：两道手写大题（均正确 ✅）

### 链表存储字符串——判断是否对称

**思路**：①遍历链表获取长度 $n$ → ②双指针，一个从头、一个从尾，逐一比较。

> 方法正确。$T=O(n)$，$S=O(1)$（如果是双链表直接头尾指针；单链表需先遍历到尾或用快慢指针找中点再逆置后半段比较）。

### 栈的括号匹配

**思路**：①左括号入栈 ②遇右括号→出栈→判断是否匹配 ③不匹配则报错 ④遍历完栈非空→报错。

> 方法正确。这是栈的**经典应用**——括号匹配 / 表达式求值 / 递归模拟三件套之一。
