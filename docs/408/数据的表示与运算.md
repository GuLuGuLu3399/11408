# 数据的表示与运算

## 核心定义

原码 是最高位表示符号的直译编码；反码 是原码到补码的过渡编码，正数不变、负数按位取反；补码 是现代计算机最常用的有符号整数表示，反码末位加 1 即得，能把 **减法统一为加法** ；移码 是补码符号位取反，常用于表示浮点数的 **阶码** ，便于比较大小。

计组主线是 **编码统一、运算统一、标志位统一** 。ALU 负责算术与逻辑运算；OF 对应 **有符号溢出** ；CF 对应 **无符号进位/借位** ；SF 反映 **结果符号位** ；ZF 反映 **结果是否为零** 。

$n$ 位补码表示范围：$-2^{n-1} \sim 2^{n-1}-1$ ；$n$ 位无符号表示范围：$0 \sim 2^{n}-1$ 。同一串二进制在不同语义下解释不同，例如 8 位 `11111111` 在无符号下为 **255** ，在补码下为 **-1** 。

$$S = A \oplus B \oplus C_{\text{in}}$$

$$C_{\text{out}} = AB + (A \oplus B)C_{\text{in}}$$

## 关键细节 / 操作步骤

1. 第一步：先判是 **有符号（补码）** 还是 **无符号** ，再确定 OF/CF 的适用语义。
2. 第二步：若问编码转换，正数原码 = 反码 = 补码；负数原码 → 反码（符号位不变，数值位按位取反）→ 补码（**反码末位加 1** ）。
3. 第三步：若问加减法，补码运算中 **符号位参与运算** ，结果仍在补码语义下解释。
4. 第四步：若问溢出判断，有符号看 **OF = 最高进位 ⊕ 次高进位** ；无符号看 **CF = 最高位进位输出** 。
5. 第五步：若问移位运算，区分 **逻辑移位（补 0）** 与 **算术移位（补符号位）** 。
6. 第六步：若问乘法， Booth 算法核心是 **判断最低两位决定加减被乘数再右移** 。
7. 第七步：若问除法，恢复余数法与加减交替法的核心区别在于 **余数为负时是否恢复后再移位** 。
8. 第八步：若问浮点表示，IEEE 754 中 $(-1)^{S} \times 1.M \times 2^{E-\text{偏置}}$ ，单精度偏置值为 **127** ，双精度为 **1023** 。
9. 第九步：若问浮点加减，对阶原则是 **小阶向大阶看齐** ，舍入方式有 **就近舍入、朝 0、朝 +∞、朝 -∞** 。
10. 第十步：若问 ALU 功能，写出它是一个统一执行 **算术运算与逻辑运算** 的组合逻辑部件。

> **⚠️ 易错辨析**
> - 补码左移不一定等于乘 2：当 **高位溢出截断** 时结果会变，必须检查字长。反例：8 位补码 `01000000`（+64）左移一位变为 `10000000`（-128），不是 +128。
> - 有符号溢出（OF）和无符号进位（CF）不是一回事：OF 判断 **补码语义越界** ，CF 判断 **无符号进位/借位** 。反例：8 位加法 `01111111 + 00000001 = 10000000`，OF=1（+127+1 溢出为 -128），但 CF=0（无进位）。
> - 算术右移要 **符号扩展** ，不能和逻辑右移混淆。反例：`10110000` 算术右移 2 位得 `11101100`，逻辑右移 2 位得 `00101100`。
> - "最高位是 1 就一定是负数"是错的，必须先看 **语义** ：无符号数最高位为 1 仍为正。
> - 移码不是独立的编码体系，它就是 **补码符号位取反** ，常用于浮点阶码以简化大小比较。

> **💡 技巧与口诀**
> 口诀：**有符号看 OF，无符号看 CF；左移先想位宽，右移先想符号；编码转三步：原→反→补（负数取反加一）** 。
> 应用场景：题目只要出现"补码运算""溢出判断""移位结果"，先按 **语义→运算→标志位** 三步走。位宽一旦固定，所有结果都要在 **该位宽内解释** ，不能随意扩大。若问机器数范围，先按字长写范围，再按语义解释。

> **📝 真题闭环**
> 题目：某 8 位补码加法运算 $x+y$ ，其中 $x=[01011010]_{\text{补}}$ ， $y=[01101000]_{\text{补}}$ ，求结果的 OF、CF、SF、ZF 及对应十进制值，并判断是否溢出。
>
> **解题思路**：
> 1. 先判语义：补码加法 → **有符号** 语义，重点看 **OF** 。
> 2. 做二进制加法：`$01011010 + 01101000 =$` **$11000010$** 。
> 3. 判 OF：最高进位 = 0，次高进位 = 1，OF = **0⊕1 = 1** → **有符号溢出** 。
> 4. 判 CF：最高位无进位输出，CF = **0** 。
> 5. 判 SF：结果最高位为 1，SF = **1（结果为负）** 。
> 6. 判 ZF：结果非零，ZF = **0** 。
> 7. 补码转十进制：`$11000010_{\text{补}} =$` **$-62$** 。
>
> 答案：OF=**1** ，CF=**0** ，SF=**1** ，ZF=**0** ，结果为 **-62（溢出）** 。验证：+90 + (+104) = +194 超出 8 位补码范围 $[-128,127]$ 。

## 强化课笔记（Day 5 补充）

### 强制类型转换与精度

| 转换方向 | 底层操作 | 精度影响 |
|---|---|---|
| `unsigned` ↔ `int`（等长） | 机器数不变，只改解释方式 | 无 |
| 短→长（无符号） | **零扩展**（高位补 0） | 无 |
| 短→长（有符号） | **符号扩展**（高位补符号位） | 无 |
| 长→短 | 截断高位，保留低位 | **可能失真/符号位改变** |
| `int`→`double` | — | **不丢精度**（52+1 位 > 31 位） |
| `int`→`float` | — | **可能丢精度**（24 位有效 < 部分 int 的有效位） |

> **浮点截断规则**：0 舍 1 入（类似四舍五入）。

### OF / CF 判定——修正你的理解

你在笔记里把 OF 和 CF 混了。这个表彻底厘清：

| 标志 | 全名 | **加法** | **减法** | 适用语义 |
|---|---|---|---|---|
| **OF** | 溢出标志 | 最高位进位 ⊕ 次高位进位 | 同加法（减法转补码加法后判） | **有符号**溢出 |
| **CF** | 进位/借位标志 | 最高位**有进位输出** → CF=1 | 最高位**有借位** → CF=1 | **无符号**溢出 |

> ⚠️ **你的纠正**：你说的"加法变号看 OF，减法变号看 CF"——**不对**。**OF 和 CF 的判定方式不因加减法而变**：
> - OF 永远看"最高位进位 ⊕ 次高位进位"（有符号溢出）
> - CF 永远看"最高位有无进位输出"（无符号溢出）
> - 加法和减法**都一样判**——因为减法在底层就是补码加法。
>
> **你真正想说的可能是**：加法中"两正得负"或"两负得正"说明有符号溢出（OF=1）；减法中"正减负得负"或"负减正得正"说明溢出。**这是 OF 的直觉判断，和 CF 无关。**

### 补码本质——你的理解对了

> 你说"负数的补码 + 绝对值 = 0 进 1，可以用取模"——**完全正确**。

补码的本质就是**模运算**：$n$ 位系统中模 $= 2^n$。
$$[-x]_\text{补} = 2^n - x$$
所以 $x + [-x]_\text{补} = 2^n = 0 \pmod{2^n}$（最高位的 1 溢出丢弃，剩下全 0）。

> 这也是"补码 = 反码 + 1"的来源：反码是"按位取反"，取反后加 1 刚好等于 $2^n - 1 - x + 1 = 2^n - x$。

### 补码与移码的关系——你的理解对了

> 你说"移码偏移值刚好是 128 所以和补码只差符号位"——**正确**。

8 位移码 = 补码**符号位取反**。
- 补码 `10000000` = -128
- 移码 `00000000` = -128（偏移 128，值 = 0 - 128 = -128）

> 移码 = 补码符号位取反 → 移码比补码大 $2^{n-1}$ → 移码值 = 补码值 + $2^{n-1}$。

### C 语言陷阱

| 陷阱 | 原因 | 结果 |
|---|---|---|
| `unsigned n=0; n-1` | 无符号数下溢 | $n-1 = 2^{32}-1$（最大值）→ 死循环 |
| `int_max + 1` | 有符号溢出 | 符号位翻转 → 变成 $-2^{31}$ |

### 高效解题方法论

**人类视角三步法**（不逐位模拟）：
1. 十进制算出**真值**
2. 检查真值是否超出该类型的表示范围
3. 分析超出后的处理（溢出翻转 / 精度截断）
