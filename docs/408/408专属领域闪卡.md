## 卡 1：减法 CF 怎么判？

### 问题
减法 `x − y` 求 CF 的规则是什么？用一句话给出 `0xFFFFFFFF − 0xFFFFFFF0`（32 位）的 CF 值。

### 答案
减法 CF 看借位：两数当**无符号数**比大小，**x < y → CF=1**，**x ≥ y → CF=0**。
该例：无符号 4 294 967 295 > 4 294 967 280，x > y → **CF=0**。

### 易错点
- 别凭"负数 / 大数相减"直觉给 CF=1，**CF 与正负号无关**，只看无符号大小。
- 硬件视角等价：减法 = `x + (~y) + 1`，CF = **最高位进位输出取反**（~C_out）。
- OF 才看符号（同号相减必不溢出）。OF 与 CF 判定**正交**。

## 卡 2：short → int 高位补什么？

### 问题
`short x = -1;`（机器数 `0xFFFF`）赋给 `int y`，y 的机器数是什么？为什么？

### 答案
**符号扩展** → `y = 0xFFFFFFFF`（仍是 −1）。高位补**符号位**：源类型最高位是几就补几。

### 易错点
- 不是补 0！**零扩展只用于无符号**（`unsigned short → int` 得 `0x0000FFFF` = 65535）。
- 判定法：**源类型有符号 → 补符号位；无符号 → 补 0**。
- 与算术 / 逻辑右移同源：**有符号保符号，无符号补 0**。
